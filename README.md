## 1. THE PROBLEM
The 2026 cloud ecosystem is fundamentally fragmented. The "Serverless vs. Edge vs. Container" debate has resulted in developers suffering from choice paralysis, vendor lock-in, and catastrophic bill shocks. 

Most AI assistants hallucinate pricing or give generic "it depends" advice. **Deploy Strategy Advisor** is a deterministic, mathematically backed Principal Cloud Architect designed natively for the Google Antigravity framework. It forces the AI to cross-reference your constraints against 14 exhaustive, up-to-date data modules before returning an opinionated architecture stack.

## 2. CODEBASE ARCHITECTURE
The system operates on a strict **Hub-and-Spoke** architecture to prevent AI context-window exhaustion and hallucination. 

<div align="center">
  <img src="assets/architecture_flowchart.png" alt="Deploy Strategy Hub-and-Spoke Architecture" width="100%">
</div>

### 2.1 The Hub (`SKILL.md`)
The `SKILL.md` file is the instruction manual. It does not contain any pricing data. Instead, it defines the **Persona** (Objective, Opinionated, Detailed) and the **Execution Workflow** (Gather Context -> Read References -> Verify Live Pricing -> Detail Limitations).

### 2.2 The Spokes (`references/`)
The `references/` directory contains 14 isolated, highly detailed markdown files acting as local databases. When you mention "React", the AI reads `01_frontend_platforms.md`. When you mention "WebSockets", it is forced to read `13_realtime_networking.md` which informs the AI that serverless edge functions cannot sustain persistent WebSockets.

### 3.1 Decision Routing Logic
- **Budget = $0**: Routes to `12_stack_recipes.md` Bootstrap Tier. Forces a combination of Cloudflare Pages/Vercel (Free), Supabase/Turso (Free), and Sentry (Free).
- **Persistent Connections / WebSockets**: Immediately flags serverless compute runtimes as incompatible. Automatically shifts backend recommendations to container hosting PaaS (Railway, Fly.io, or Render) using `02_backend_platforms.md`.
- **Database Scalability**: Evaluates Neon for serverless PG scaling or Supabase for BaaS integrations based on `03_database_services.md`.
- **Global Latency Requirements**: Routes to Deno Deploy or Cloudflare Workers in `05_edge_cdn_serverless.md` to distribute compute to V8 isolates globally.

## 4. KNOWLEDGE BASE REFERENCE REFERENCE GUIDE (THE 14 NODES)

Every recommendation generated by this skill relies on the following 14 data files in the `references/` folder. Here is the exact scope and content outline for each file:

### 🚀 Compute & Hosting Nodes
*   **`01_frontend_platforms.md`**: Outlines Vercel, Netlify, Cloudflare Pages, Firebase Hosting, and AWS Amplify. It covers the specific limitations of Hobby tiers, build limits, Next.js optimization compatibility, and edge middleware pricing constraints.
*   **`02_backend_platforms.md`**: Covers Railway, Render, Fly.io, Koyeb, Zerops, Qovery, and Heroku. It details container-based compute costs, Nixpacks/Docker build limits, private networking layers, and micro-VM scaling logic.
*   **`06_cloud_providers.md`**: Covers the hyper-scale cloud options: AWS, GCP, Azure, DigitalOcean, Hetzner, Vultr, and Linode. It analyzes managed Kubernetes pricing, ingress/egress cost traps, and raw virtual machine performance-to-cost metrics.
*   **`05_edge_cdn_serverless.md`**: Analyzes Cloudflare Workers, Deno Deploy, Fastly Compute, and AWS Lambda@Edge. It details V8 isolate CPU time constraints (e.g., the 10ms/50ms execution limit), global edge distributions, and Node.js standard library polyfill limitations.
*   **`07_self_hosted.md`**: Guides users on self-hosting their own PaaS. It evaluates Coolify, Dokploy, CapRover, Dokku, Appwrite, and PocketBase. It provides hardware sizing tables (RAM/CPU limits) for running these control planes on cheap VPS platforms.

### 💾 Data, Caching & Real-Time Nodes
*   **`03_database_services.md`**: Detailed mapping of Supabase, Neon, PlanetScale, Turso, and Upstash. Covers connections limits, storage quotas, backup retenion periods, and PostgreSQL extension support.
*   **`13_realtime_networking.md`**: Covers Ably, Pusher, Liveblocks, LiveKit, NATS, Kafka, and native WebSockets. Explains pricing curves for connection-minutes, pub/sub limits, and latency implications for collaborative cursors.

### 🔒 DevSecOps, Observability & Security Nodes
*   **`04_cicd_devops.md`**: Analyzes GitHub Actions, GitLab CI/CD, CircleCI, Buildkite, Woodpecker CI, ArgoCD, FluxCD, Terraform, OpenTofu, and Pulumi. It outlines parallel runner pricing, caching limits, and IaC deployment lifecycles.
*   **`08_monitoring_security.md`**: Maps Datadog, Sentry, Grafana Cloud, Axiom, and BetterStack. It details APM, metrics retention limits, logging ingestion rates, and strategies to prevent billing spikes.
*   **`14_security_secrets.md`**: Covers Doppler, Infisical, Vault, Trivy, Snyk, and Gitleaks. Explains how to securely inject environment variables into containers and set up automated container image vulnerabilities scanning.

### 🧠 Architecture, Calculator & Recipe Nodes
*   **`09_ai_ml_specialized.md`**: Details Modal, RunPod, Replicate, Together AI, and HuggingFace. Outlines exact hourly/per-second GPU pricing (A100, H100, RTX4090) and cold start mitigation for custom LLMs.
*   **`10_architecture_patterns.md`**: Framework detailing Monoliths vs. Microservices vs. Serverless, sizing them based on traffic volumes and team sizes.
*   **`11_pricing_calculator.md`**: Formulates mathematical cost calculations for CPU, memory, storage, and bandwidth.
*   **`12_stack_recipes.md`**: Pre-configured architecture blueprint sets for bootstrap ($0), startup ($15), scale-up ($50-$200), and enterprise ($500+) tiers.

git clone https://github.com/kunal-gh/deploy-skill.git ~/.gemini/config/skills/deploy-strategy

### 6.2 Triggering the AI
You do not need to invoke special commands. Simply ask your Antigravity agent a deployment question. To get the best results, provide the required context variables:

**Optimal Prompt Structure:**
> "I am building a **[Tech Stack]** for a **[Project Type]**. I expect **[Traffic Limit]** and my budget is **[Budget]**. Do I need **[Special Requirements]**? What is the best deployment strategy?"

**Example Real-World Prompt:**
> "I'm building a SaaS MVP with Next.js, Postgres, and Redis. We need collaborative cursors (WebSockets). My budget is $50/mo total. We expect 10,000 MAU. What should we use?"

## 7. CONTRIBUTION GUIDELINES

The cloud landscape changes rapidly. When Railway removes a free tier, or Vercel changes bandwidth pricing, this repository must be updated.

We rely on the open-source community to keep the 14 reference nodes mathematically accurate.

### 7.1 Fork and PR Flow
1. **Fork the Repository:** Create your own copy of the repo on GitHub.
2. **Local Setup:** Clone the fork directly into your local Antigravity customizations folder (`~/.gemini/config/skills/deploy-strategy`).
3. **Edit a Spoke File:** Update the specific `.md` file inside the `references/` directory. Keep pricing tables aligned and details factual.
4. **Local Verification:** Run a query against your Antigravity agent using the local skill to verify it parses your changes correctly.
5. **Commit and PR:** Push your changes to a branch and submit a PR to the main repository.
